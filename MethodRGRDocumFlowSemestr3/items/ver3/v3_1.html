Об’єктно-орієнтоване програмування

 
 
 

Версія 3. Методи класів: аксесори, ітератори, утілити, предикатні функції. Діаграми класів, діяльності, послідовності.

Мета:

  • ознайомитись з різними варіантами методів класу. Доповнити програму прототипами та реалізаціями методів згідно з UML-діаграмами.

Теоретичні відомості

  • Інкапсуляція
  • Аксесори
  • Предикатні функції
  • Діаграма послідовності та діаграма діяльності
Визначають такі категорії функцій-елементів класу:
  • спеціальні функції - конструктори, деструктори;
  • функції доступу (аксесори), що повертають значення закритих даних-елементів класу;
  • функції установки (аксесори), що встановлюють значення закритих даних-елементів класу;
  • функції-утілити – закриті функції, що викликаються з відкритих функцій для опису поведінки об’єкту, яка здійснюється під впливом закритих елементів-даних;
  • предикатні функції, які визначають істинність чи хибність певних умов;
  • операторні функції перетворення стандартних операцій і операторів для застосовування до типів даних користувача;
  • функції-ітератори, які реалізують доступ до усіх даних-елементів класу у визначеній послідовності;
  • функції-селектори, які зчитують стан об’єкта, але не змінюють його;
  • обробники подій, які є реакцією на певні події (натиск кнопки клавіатури, миші, вибору команди меню тощо).

Інкапсуляція

Принципи інкапсуляції свідчать, що:
  • Внутрішні структури даних, які використовуються в реалізації класу, не можуть бути доступні об’єкту класу безпосередньо.
  • Жодний об’єкт не може безпосередньо зчитувати або змінювати стан будь-якого іншого об’єкта.
  • Доступ до атрибутів іншого об’єкта здійснюється через методи його зовнішнього інтерфейса.
  • Такий доступ можливий, тільки якщо сам об’єкт це дозволяє.
Принципи інкапсуляції означають, що:
  • об’єктно-орієнтована система здійснює приховування інформації, яке включає приховування самих даних і виконання операцій над ними;
  • інкапсуляція збирає разом дані та функції, які обробляють ці дані.
Отже, все, що інкапсульовано, можна розглядати як єдине та приховане від користувача класу.

Переваги інкапсуляції:
  • Можливість перевірки правильності будь-яких значень, які надаються змінним-членам, що запобігає помилкам програмування.
  • Можна вільно змінювати спосіб зображення даних-членів, не змінюючи інші частини програми, які використовують клас (зберігаючи інтерфейс класу, можна вільно змінювати його реалізацію).
  • Інкапсуляція виключає залежність користувача класу від специфічного внутрішнього зображення даних.
Доступ до елементів-даних класу:
  • Реалізація доступу до змінних-членів здійснюється з урахуванням специфікатора доступу.
  • По умовчанню всі змінні і функції класу визначені як закриті (private).
  • Це значить, що вони можуть використовуватися тільки усередині функцій-членів самого класу.
  • Специфікатор доступу застосовується до всіх членів, які розміщені після нього в означені класу, поки не зустрінеться інший специфікатор доступу.
  • Коли всі члени класу відкриті, доступ до них можливий через об’єкт з використанням оператора крапки «.» .
  • Можна створити екземпляр класу оператором new, а потім використовувати покажчик на екземпляр класу для доступу до відкритих змінних-членів класу.
  • Якщо об’єкту класу заборонено прямий доступ до змінних-членів, клас має надати альтернативний спосіб отримання їх значень.
  • Реалізація доступу до закритої частини класу.

Аксесори

Дії аксесорів:
  1. Повернути значення закритих елементів даних:
    1. такі функції можуть не мати параметрів і повертають значення через оператор – return;
    2. якщо слід повернути з функції більше одного значення, такі функції можуть приймати в якості параметрів покажчики чи посилання, через які повертатимуть значення.
  2. Встановити значення закритих елементів даних:
    1. такі функції приймають в якості параметрів значення, що потрібні для присвоєння елементам даних;
    2. якщо встановлено більше одного значення, то його повертають через покажчики чи посилання;
    3. якщо встановлено одне значення, то його повертають через return.
[up]

Предикатні функції

Предикатні функції служать для перевірки істинності чи хибності умов та повертають, булеві значення true або false.

Предикат - це функція з множиною значень {0,1} («хибність» та «істина») визначена на множині M = M1 × M2 × ... × Mn так, що кожен набір елементів множини M характеризується або як "істинний", або як "помилковий". Предикат можна зв'язати з математичним відношенням: якщо множина (М1, М2,.., Мn) належить відношенню, то предикат повертатиме на ній 1 (true).

Дії аксесорів: У будь-якого потоку є набір прапорців, за допомогою яких можна стежити за станом потоку.

Є чотири предикативні функція-члени:
  1. eof() повертає true, якщо досягнутий кінець файлу:

    if ( inOut.eof() ) // відмінно: усе прочитано ..


  2. bad() повертає true при спробі виконання некоректної операції, наприклад, при установці позиції за кінцем файлу. Зазвичай це свідчить про те, що потік знаходиться в стані помилки.


  3. fail() повертає true, якщо операція завершилася невдало, наприклад, не вдалося відкрити файл або переданий некоректний формат введення:

    ifstream iFile( filename, ios_base::in );
    if ( iFile.fail() ) // не вдалося відкрити error_message( .. );


  4. good() повертає true, якщо усі вище перелічені умови помилкові:

    if ( inOut.good() )
